Practical intro

In this first practical we will fork a facebook-mashup example from the github account and customize it. We have posed several challenges on different skill levels, please feel free to interpret these as you please and create something beautiful!

Prerequisites

  • Install Chrome browser
  • Install and Setup Git
  • Fork and git clone the Capgemini-HTML5 / Sessions repository
  • Install notepad++ (basic editor) or Aptana Studio (advanced editor)

Facebook Mashup code

When forked, please take a look at Willems version of the Mashup
  • index.html - contains HTML5 markup code
  • css/user.css - contains the custom CSS3 styling
  • js/script.js - contains the custom Javascript code

Challenges

Beginner:
  1. Customize the mashups HTML (e.g. add a couple of tags, change some text, ...)
  2. Customize the spinner CSS (e.g. adjust the speed, change the transitions, ...)
  3. Customize the Javascript (e.g. add a fade to the spinner loading, customize the rendered data html, ...)
Intermediate
  1. Get your own access token on Facebook
  2. Use your own access token to retrieve liked facebook pages instead of friends (hint: Facebook Graph documentation)
  3. Display Facebook pages in a visual manner (list, text, canvas, images, ...)
  4. Push your commits back to a new folder (your own folder, next to Willem and Peters) the git repository
Expert:
  1. Use HTML5 canvas tag to display the retrieved facebook data in an interesting way.
  2. Use local storage API to store/cache the retrieved data and make the app work offline.
